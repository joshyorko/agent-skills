# Shape Up Cycle

Use this when a shaped pitch needs cycle/cool-down/betting-table language or
when an existing-product refresh is large enough to become a bet.

Do: keep appetite fixed and scope variable, name the bet, define no-gos and the
circuit breaker, split build work into scopes, and communicate progress around
unknown-to-known-to-done movement instead of task churn.

Avoid: sprint-planning language, elastic deadlines, backlog dumps, automatic
extensions, and status artifacts that interrupt the team to ask what a scope
already shows.
